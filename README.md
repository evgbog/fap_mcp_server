# MCP-сервер для Платформы Форсайт

## ✨ Возможности

- Создание ИИ-агентов с возможностью выполнения запросов к многомерным данным на естественном языке
- Гибкий выбор (подключение) многомерных источников (кубов) для ИИ-анализа
- Возможность переопределить метаданные источника (наименования/описание источников, измерений, мер, форматы данных, единицы измерения и др.)

## 🚀 Установка

### Расположение компонентов

1. **`python/`** — реализация MCP-сервера на языке Python с использованием библиотеки FastMCP.
2. **`fap_component/`** — компонент для Платформы Форсайт (обработка OLAP-запросов, мета-описания, интерфейс настройки).
3. **`examples/`** — примеры, включая системный промпт агента **«BI аналитик»**.

### Установка компонента в репозиторий Платформы

Установите обновление `fap_mcp_server_1.x.x.pefx` из папки `fap_component`. В процессе установки в папке компонентов будет создана папка `ИИ. MCP сервер ФАП`, в которой будет установлена БД `DB_MCP_META`, указывающая на БД метаданных текущего репозитория, созданы 3 таблицы для хранения метаданных, и ряд дополнительных объектов.

Ключевые объекты:
- Сервисный модуль для MCP-сервера (`MOD_MCP_SERVER`)
- Веб-форма Редактор источников для ИИ-анализа (`FRM_MCP_SETUP`)

### Установка MCP-сервера

Реализация сервера на Python находится в файле `fap_mcp_server.py`. Предполагаем, что сервер будет развернут на Linux-сервере в режиме streamable-http. Создайте виртуальную среду, установите необходимые зависимости, при необходимости оформите запуск в виде службы. По умолчанию используется порт 8000, измените при необходимости.

Дополнительно в папке имеется реализация MCP-сервера с встроенной генерацией синтетических данных - `test_mcp3.py`. Мы использовали его для тестирования, он не требует наличия Форсайт-части.

## 🛠 Настройка

### Генерация токена для выполнения кода на стороне Платформы

Реализация MCP-сервера на Python использует API Платформы для выполнения Fore-кода `ForeExecAtomic`:
https://help.fsight.ru/ru/mergedProjects/ppsomservice/operations/repository/foreexecatomic.htm

Для работы этой функции необходимо в `Менеджере безопасности` создать технического пользователя с правами, позволяющими читать метаданные компонента и данные многомерных кубов, после чего сгенерировать соответствующий токен. Порядок генерации токена описан по ссылке выше.

### Настройка Python-скрипта, реализующего сервер MCP

В коде MCP-сервера задайте значение 4 строковых параметров:

- `SERVICE_URL` - сервисный URL (тот же, что указан в параметре `serviceUrl` файла `config.json` веб-приложения)
- `FORE_EXEC_TOKEN`  - токен, сгенерированный для выполнения Fore-кода
- `METABASE_ID` - идентификатор метабазы
- `MCP_MODULE_KEY` - ключ сервисного модуля `MOD_MCP_SERVER` (числовое значение в виде строки). Значение ключа можно посмотреть в окне "Свойства" данного модуля.

### Настройка ИИ-агента (на примере Cherry Studio)

Вы можете использовать любой подходящий ИИ-клиент для работы, в наших примерах мы использовали популярный бесплатный клиент `Cherry Studio`.

1. В настройках `Провайдеры моделей` подключите LLM. На момент публикации (июнь 2026) мы использовали LLM-модели Qwen 3.6 27b или Qwen 3.6 35b A3B - рекомендуем начать с них.
2. В настройках `MCP` создайте MCP сервер, тип укажите `Потоковый HTTP`, укажите URL сервера `http://my_server:8000/mcp`
3. Включите опцию `Длительный режим работы`.
4. Если все настройки выполнены корректно и клиент "увидел" сервер, у вас станет доступна закладка "Инструменты", где должны появится список из 4-х инструментов. Проверьте, что для всех инструментов включены опции `Включить инструмент` и `Автоматическое одобрение`.
5. В разделе `Главная` - `Ассистент` создайте ассистента `BI-аналитик`.
6. В разделе `Настройки модели` сопоставьте ему ранее настроенную LLM-модель и задайте `Температура = 0.2`, `Top-P = 0.8`.
7. В разделе `Настройки промптов` задайте системный промпт ассистента (пример приведен в `system_prompt` в папке `examples\`).
8. Рекомендуем выключить режим `thinking` модели. Для этого создайте пользовательский параметр `chat_template_kwargs` типа `JSON` со значением ```{
    "enable_thinking": false
}```

На этом настройка клиента завершена.

### Настройка доступных ИИ-ассистенту источников

Для настройки доступных ИИ-ассистенту источников их необходимо добавить в специально созданной для этого интерфейсной веб-форме.

1. В навигаторе Платформы откройте веб-форму `FRM_MCP_SETUP`.
2. В выпадающем списке `Выберите источник данных` укажите многомерный куб, который нужно подключить к анализу. Нажмите `Добавить` (добавлять можно только кубы, у которых для всех фактов в мастере куба включена агрегация `Сумма`, без этого извлечение данных может работать не корректно).
3. В разделе `Дополнительные метаданные` отредактируйте наименования и описания источников, измерений, мер.
4. Для мер дополнительно можно указать формат вывода мер, тип данных и единицы измерения. Информация о том, какие значения тут можно указывать, приведена в сервисном модуле `MOD_MCP_SERVER`.
5. Если возникла необходимость временно отключить источник для доступа ИИ-ассистента, воcпользуйтесь флагом `Активный источник` (снимите его).
6. От того, насколько подробно будут введены наименования и описания источников, мер и измерений, зависит способность ИИ-ассистента максимально точно ответить на вопрос пользователя.

### Примеры текстовых запросов, на которые ИИ-ассистент может отвечать

- Доступные источники?
- Покажи доступные измерения и меры.
- Общая выручка за всё время?
- Выручка в разрезе групп товаров?
- Выручка за январь 2020?
- Общая выручка за 1 декаду января 2020?
- Выручка по дням за 1 декаду февраля 2020?
- Выручка по месяцам за 1 квартал 2020 по группам товаров "Техника для кухни" и "Техника для дома"
